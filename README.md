## 🚀 Technology Stack

- **Vue 3**: Using the Composition API with `<script setup>` syntax for more concise and readable components
- **Vite**: Next-generation frontend tooling for faster development and optimized builds
- **TypeScript**: For type safety, better IDE support, and improved developer experience
- **Pinia**: State management solution that's intuitive and type-safe
- **Vue Router**: Client-side routing with support for lazy loading components
- **Tailwind CSS**: Utility-first CSS framework for rapid UI development
- **ofetch**: Modern and efficient HTTP client for API communication
- **Vitest**: Unit testing framework optimized for Vite
- **Cypress**: End-to-end testing framework for reliable application testing

## 🔧 Development Setup

### Recommended IDE Setup

[VS Code](https://code.visualstudio.com/) + [Volar](https://marketplace.visualstudio.com/items?itemName=Vue.volar) (and disable Vetur).

### Type Support for `.vue` Imports in TS

TypeScript cannot handle type information for `.vue` imports by default, so they are shimmed as a generic Vue component type. In most cases this is sufficient, but if you need precise prop types in `.vue` imports, enable the Take Over Mode by running `Volar: Switch TS Plugin on/off` from VS Code command palette.
